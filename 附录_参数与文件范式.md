# 附录:参数说明 与 各产物文件内容范式

> 论文附录用。A 部分解释运行/agent 各项参数(含 codex 的 model / reasoning / sandbox);B 部分逐文件给出内容范式(schema)。标注 ✅=代码现存 / 🟡=已设计待落地。

---

## A. 参数说明

### A.1 出题/候选 agent 调用参数(`agent/client.py`)

统一入口:`run_headless(prompt, cwd, allowed_tools, engine, timeout=1200, model, add_dirs, reasoning_effort)`。

| 参数 | 取值 | 含义 |
|---|---|---|
| `engine` | `claude` / `codex` / `stub` | 用哪个无头 agent;`stub`=离线占位(不调模型) |
| `model` | 如 `gpt-5.5`(codex)/ claude 模型名 | codex 走 `-m <model>`,claude 走 `--model <model>` |
| `reasoning_effort`(codex) | `low < medium < high < xhigh`,**默认 `high`**(`CODEX_REASONING_EFFORT`) | 推理深度(配置覆写,非换模型):`-c model_reasoning_effort=high`。gpt-5.5 原生默认 medium;`high` 适合多仓吞吐,`xhigh` 单仓最高质量但 ~2–3× 慢(~50min/仓) |
| sandbox/审批(codex) | `--sandbox workspace-write` | 无人值守:工作区内自由读写、免逐步审批("**auto**"即此)。需逃逸沙箱跑 build/SUT 时换 `--dangerously-bypass-approvals-and-sandbox` |
| permission(claude) | `--permission-mode acceptEdits` | 自动接受文件写入(无人值守) |
| `allowed_tools` | 如 `["Read","Grep","Glob","Write","Bash"]` | claude 用 `--allowedTools` 强制;**codex 是 advisory**(实际由 sandbox 管) |
| output(claude) | `--output-format json` | 机器可解析输出 |
| `add_dirs` | 路径列表 | 额外可读/写目录(如原仓源码) |
| `timeout` | 默认 1200s;Stage0 容器化 1800s;候选生成 2400s | 子进程超时 |
| 其它(codex) | `--cd <cwd> --skip-git-repo-check --color never`,stdin=DEVNULL | 工作根 / 跳过 git 检查 / 无色 / 防 stdin 串入 prompt |

> **你举的 "codex -gpt5.5 -auto -high" 对应**:`-m gpt-5.5`(模型)+ `--sandbox workspace-write`(auto 免审批)+ `-c model_reasoning_effort=high`(推理深度 high)。

### A.2 运行入口参数

| 入口 | 参数(默认) | 说明 |
|---|---|---|
| `run_dataset.py`(整套) | `--manifest`(必填)·`--agent`(claude)·`--jobs`(8)·`--from/--to`(0/7)·`--only`(子集) | 跑整个数据集,仓库级并行 |
| `orchestrate.py`(单仓) | `--repo`(必填)·`--manifest`·`--agent`(stub)·`--from/--to`(0/7) | 单仓、单阶段范围 |
| `chuti.py` | `--repo`/`--all`·`--agent`(stub)·`--from/--to`(0/7)·`--jobs`(8) | 顶层 CLI |
| `run_candidate.py`(B+C) | `--repo`(必填)·`--candidate-engine`(codex)·`--candidate-id`(cand1)·`--model` | 驱动候选答题 + 判卷 |

### A.3 流水线参数

| 参数 | 默认 | 来源 | 含义 |
|---|---|---|---|
| `quota` | 20 | manifest.defaults / 每仓 | 每模块最少测试数 |
| `max_rounds` | 3(gron 用 5) | manifest / orchestrate | Stage 5 fill_quota 最多轮数 |
| `runtime_mode` | docker | manifest.defaults | docker / local |
| `coverage.mode` | 按语言 preset(go/lcov/coveragepy/none) | config.py PRESETS | 覆盖率采集器 |
| 双跑定性 | 每输入跑 2 次 | stage5_loop | 不一致即丢(`dropped_nondeterministic`) |
| Stage7 mutation | 仅 `coverage==go` | stage7_verify | 区分度关(非 go 仓暂无) |

### A.4 维度配置参数(🟡 ISO v3 设计)

| 文件 | 关键参数 |
|---|---|
| `perf_policy.yaml` | `repetitions=3`、`suite_time_budget_s=600`、`L95_upper_s=30`、`mem_peak_upper_bytes=4GiB`、子特性权重 |
| `portability_policy.yaml` | `eval_image`、`build_timeout_s=600`、`case_timeout_s=30`、`network_mode`、profiles `P0–P4`、子特性权重 |
| `dependency_catalog.yaml` | 各生态允许依赖集合 + `system:` |
| 静态规则文件 | 每条 `{id, severity, penalty}` + 封顶(`same_rule_cap/same_file_cap/subchar_cap`) |

## B. 各产物文件内容范式（逐个）

### B.1 输入 / 全局配置

**`dataset/*.manifest.json`** ✅ — 数据集清单(你唯一要填)
```jsonc
{ "dataset": str, "defaults": {"quota": 20, "runtime_mode": "docker|local"},
  "repos": [ { "id", "scenario", "scenario_type": "cli|service|pipeline", "language",
    "source": {"kind":"git|path","url?","ref?","path?"},
    "runtime?": {"mode","dockerfile?","image?","cover":{"mode":"go|lcov|coveragepy|none","collect?"}},
    "launch?": [...], "service?": {"port","health","env","start_timeout_s"},
    "pipeline?": {"in","out","cmd","compare"}, "dependencies?": [{"kind","env","db_name?"}],
    "quota?", "max_rounds?" } ] }
```

**`configs/<repo>.json`** ✅ — 单仓简版(back-compat):`{repo_id, launch[], drafts, out_dir, coverage}`

**`dataset/metrics-table.json`** ✅ — 用户提供的 NFR 指标:`[{id, name, description, target?}]`

**`configs/perf_policy.yaml` / `portability_policy.yaml`** 🟡 — `{version, mode, execution{...}, scoring{权重}, profiles?[], static_scan{...}}`

**`configs/static_perf_rules.yaml` / `static_portability_rules.yaml`** 🟡 — `{rules:[{id, severity, penalty}], explanations:{id: 模板文本}}`

**`configs/dependency_catalog.yaml`** 🟡 — `{python:[pkg==ver], rust:[...], npm:[...], go:[...], maven:[...], system:[...]}`

### B.2 schemas/*.json

| 文件 | 范式 |
|---|---|
| `testcase.schema.json` ✅ | `{id, modules[], scenario_type, kind, input{}, assertions[{field, class:"exact\|normalized\|invariant\|ignored"}]}` |
| `repo-model.schema.json` ✅ | `{repo_id, scenario_type, language, external_deps[{kind:"env-download\|controllable-network\|uncontrollable-realtime"}], determinism_hazards[]}` |
| `modules.schema.json` ✅ | `{modules[{id:^M[0-9]+$, name}]}` |
| `candidate_manifest.schema.json` 🟡 | `{language_profile, project_type, dependencies{eco:[...]}, entry{command[]}, readiness?{path,expected_status}}` |
| `performance_report / portability_report.schema.json` 🟡 | 见 ISO v3 报告格式 |

### B.3 out/<repo>/ 产物(逐文件)

**`00_baseline/baseline.json`** ✅ — ingest 结果
```jsonc
{ "repo_id","scenario_type","language","runtime_mode","coverage",
  "build_attempts", "image"|"cover_binary", "baseline_test?", "smoke" }
```

**`00_runtime.json`** ✅ — agent 写的"怎么跑"
```jsonc
{ "scenario_type", "service?":{"port","health","env"}, "pipeline?":{"in","out","cmd"},
  "dependencies":[{"kind","env"}], "smoke":[...], "notes" }
```

**`01_repo-model.json`** ✅ — 理解(题面)
```jsonc
{ "repo_id","scenario_type","language", "build":{"toolchain","cmd"}, "test_cmd",
  "coverage":{"tool","granularity"}, "summary", "public_surface":[...],
  "external_deps":[{"kind","what","handling"}], "determinism_hazards":[...],
  "stateful": bool, "rewritable_languages":[...] }
```

**`02_cli-contract.json`** ✅ — 锁定契约(cli;题面核心)
```jsonc
{ "contract_type":"cli", "binary", "usage",
  "actions":{"precedence":[...], "<action>": doc},
  "flags":[{"long","short","alias?","type","doc"}],
  "input_resolution":{...}, "exit_codes":{"<n>": meaning},
  "io_contract":{"stdin","stdout","stderr","color"},
  "candidate_delivery":{"must":[...], "network_inputs":[...]} }
```
- `02_contract.openapi.json`(service)= OpenAPI 3;`02_contract.io.json`(pipeline)= `{in,out,cmd,exit_codes}`

**`03_modules.json`** ✅
```jsonc
{ "modules":[{"id":"M1","name","user_value","ops":[...],"flags":[...],
              "input_contract":[...],"io_contract":[...],"exit_codes":[...]}],
  "coverage_of_contract", "quota":{"target_per_module":20} }
```

**`03_user-stories.json`** ✅
```jsonc
{ "stories":[{"id","modules":[...],"source",
   "prose":{"actor","precondition","trigger","expected"},
   "code":["<契约调用>","observe(...)"]}] }   // code 不含期望值
```

**`04_nfr-probes.json`** ✅
```jsonc
{ "probes":[{"metric_id","applies_to":[...],
   "method":"capability_gate|load|fault|static","probe","pass_if",
   "env":{"cpus","memory"},"fairness","origin":"implemented|added_requirement",
   "scoring":"in_functional|separate"}],
  "not_observable":[{"metric_id","reason"}] }
```

**`05_tests/<id>.json`** ✅ — 测试用例(隐藏,判卷集)
```jsonc
{ "id","modules":[...],"scenario_type","kind":"intent|incident","note",
  "input": {"argv":[...],"stdin|stdin_b64","files":{}}      // cli
          | {"steps":[{method,path,headers?,body?,bind?}]}  // service
          | {"files":{...}},                                 // pipeline
  "observed": {预览},
  "assertions":[{"field":"exit|stdout|stderr|file:<p>|http:N:...","class","rule?","value?"}] }
```

**`05_coverage-ledger.json`** ✅
```jsonc
{ "summary":{"repo_id","scenario_type","tests_emitted","per_module":{},
   "needs_review":[...],"untagged_tests","dropped_bad_spec","dropped_nondeterministic",
   "quota","intent","incident","pct_statements","uncovered_blocks","coverage_climb":[[n,pct]]},
  "uncovered_sample":[...] }
```

**`05_input-drafts.json` / `_drafts_round.json`** ✅ — agent 产的输入草稿
```jsonc
[ {"id","modules","scenario_type","kind","note","input":{...},
   "assert_spec":[{"field","class","rule","note"}]} ]   // 仅断言"类",无 value
```

**`06_adversarial.json`** ✅
```jsonc
{ "open_critical": int, "findings":[{"test_id","issue","severity":"critical|major|minor","fix"}], "notes?" }
```

**`07_verify.json`** ✅
```jsonc
{ "high_water":{"passed","total","rate","failures":[id...]},
  "mutation":{"kill_rate","killed","scored_mutants","details":[{mutant,result,expected,...}]} }
```

**`STATUS.json`** ✅ — 断点续跑:`{ "0".."7": "pass" | "fail: <reason>" }`

**`06_performance/run_report.<cid>.json`** 🟡 / **`07_portability/run_report.<cid>.json`** 🟡(ISO v3)
```jsonc
{ "mode","candidate_id","subcharacteristics":{<sub>:{"dynamic","static_penalty","static","score"}},
  "static_scan":{...}, "final":{"formula","<Dimension>Score"} }
```
**`static_scan_report.<cid>.json`** 🟡 — `{findings:[{rule_id, layer:"mechanical|llm", subcharacteristic, file, line_start, line_end, evidence, confidence?, explanation}], by_subcharacteristic:{}}`

**`07_exam/`** ✅ — 打包(拆分)
- `candidate/`:`{02_*-contract, prompt.md, generation_language.txt, 功能模块文档.md, 用户行为示例文档.md}`(给考生)
- `grader/`:`{cases/*.json, _assert.py, conftest.py, test_blackbox.py, pytest.ini, measurable_metrics.json, rewritable_languages.json, _dropped_poisoned/}`(判卷,pytest 化)
- `package.json`:`{grader_tests, measurable_metrics, rewritable_languages, generation_language}`

**`exam-packages/<repo>/`** ✅ — 可分发考卷:候选 SPEC + `Dockerfile` + `vendor/`(参考实现,grader 侧) + grader

**`out/dataset-report.json`** ✅ — 数据集汇总
```jsonc
{ "dataset","total","completed",
  "repos":{"<id>":{"ok","status":{...},"stage5":{...},"stage7":{...}}} }
```
