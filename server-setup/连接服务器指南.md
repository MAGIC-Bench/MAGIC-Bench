# 连接服务器（容器）指南

平台：**Cube-Studio**。你创建的「node」实际是一个 Ubuntu 容器，要通过**跳板机**SSH 进去。

---

## 第一步：拿到你自己的端口和密码（关键）

文档里出现的 `11021 / tt1234!!` 之类**全是示例**，不是你的。你自己的端口和密码在你 notebook 里的一个文件中：

1. 浏览器打开 http://172.16.108.241/frontend/ ，用 **NETID** 登录
2. 顶部 **在线开发 → notebook**，确认你的 notebook 状态是 **Running**（不是 Running 就点名字/等它起来）
3. 点 notebook 名字进入 Jupyter
4. 左侧文件树根目录（显示为 `/`，其实是 `/mnt/<你的NETID>`）里有一个名为 **「ssh连接」** 的文件，双击打开
5. 文件第 2 行注释就是你的连接信息，形如：
   ```
   # ssh root@cubestudio_container -p <你的端口> 登录密码: <你的密码>
   ```
   记下这个**端口**和**密码**。

文件里还附了一段现成的 ssh config（jumpServer + cubestudio_container 两段），可以整段复制。

---

## 第二步：配置本地 ~/.ssh/config

把 [ssh_config.template](ssh_config.template) 的内容追加到本地 config 文件：

- Windows 路径：`C:\Users\<你的用户名>\.ssh\config`
- 把 `cubestudio_container` 段里的 **Port** 改成你第一步拿到的端口

（或者直接复制 notebook 里「ssh连接」文件中的那两段，效果一样。）

---

## 第三步：连接

### 校内 / 校外都通用（推荐，走跳板机）

配好 config 后：
```bash
ssh cubestudio_container
```

不想改 config 的话，一行命令等价：
```bash
ssh -J cubestudio@172.16.108.241 -p <你的端口> root@172.16.108.241
# -p 后面是你的【容器端口】
```

连接过程会问两次密码（如果你还没配免密）：
- **跳板机 cubestudio 用户密码**：`cubestudioOc1.`（注意结尾有个点）
- **容器 root 密码**：第一步「ssh连接」文件里看到的那个密码

> ⚠️ 不要直接 `ssh cubestudio@172.16.108.241` 登跳板机，会被拒绝。必须用上面的跳板（ProxyCommand / -J）方式连到容器。

### 为什么必须走 22 端口的跳板机？

校外通过 VPN（非科研区）访问防火墙时只放通 `22/80/443/3389`。容器端口（如 11021）不在白名单里，直连不通；跳板机在 **22** 端口，所以无论校内校外都能连。**校外记得先连 VPN**：https://inc.sysu.edu.cn/service/vpn

## 可选：配置免密登录跳板机

如果跳板机密码反复输错、或想免密，把官方文档 **第 24 页** 的 PowerShell 脚本复制到 PowerShell 里运行一次。
它会把 `cubestudio` 用户的私钥写到 `C:\Users\<你的用户名>\.ssh\cubestudio_rsa`（UTF-8 编码），之后 config 里的
`IdentityFile ~/.ssh/cubestudio_rsa` 就能自动免密跳转。

> 那段私钥太长、且 OCR 容易抄错一个字符就失效，建议直接从 PDF 第 24 页复制原文。需要的话我可以帮你把它转成一个 `.ps1` 脚本放进本文件夹。

## 关于「硬盘分配了多少」

**文档里没有写明给每个 node 分配多少硬盘。** 文档只说明了：

- 你的**持久化目录**是 `/mnt/<你的NETID>`，挂的是共享存储卷（`kubeflow-user-workspace`）。
  **只有持久化目录里的数据不会随容器删除/reset 而丢失**，其它目录的数据容器一删就没。
- 新建 notebook 时填的 `内存申请(10G)`、`cpu申请(10)` 是**内存和 CPU**，不是硬盘。
- 使用须知只要求「注意控制占用的存储空间，不用的及时删」，没给固定配额。

**想知道实际可用空间**，连上容器后直接看：
```bash
df -h /mnt/$(whoami)     # 你的持久化目录可用空间
df -h /                 # 容器临时层空间
du -sh /mnt/$(whoami)/* # 看自己已经占了多少
```
**想知道分配/配额上限**，按使用须知第 0 条直接问群管理员（助理 梁竞冀 / 老师 王伟成）最准。
