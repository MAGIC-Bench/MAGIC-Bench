# 实验服务器配置工作区

本文件夹用于「未来实验服务器」的配置工作，与上层仓库内容无关。

服务器平台是 **Cube-Studio**（一个 ML 算力平台）。你不是直接 SSH 到一台裸机，而是：
登录 Web 控制台 → 新建一个 **Notebook**（本质是一个 Ubuntu 容器，就是你说的「node」）
→ 通过**跳板机**SSH 进这个容器 → 在容器里装依赖 → **保存成镜像**。

## 整体计划

1. **连上第一台服务器（容器）** ← 你现在在这一步，见 [连接服务器指南.md](连接服务器指南.md)
2. 在容器里安装依赖、配好运行环境
3. 把容器 **保存成镜像**（notebook 行的「镜像保存」按钮 → 等 pod 状态 `Succeeded`
   → 在「详情」里看到镜像名，形如 `172.16.108.243:8080/cube-studio/jupyter-user:<netid>-<id>`）
   ⚠️ **务必自己记下镜像名**，notebook 删了之后就看不到了

## 关键事实速查

| 项目 | 值 |
| --- | --- |
| Web 控制台 | http://172.16.108.241/frontend/ （用**学校 NETID** 登录，首次登录自动注册） |
| 跳板机 | `172.16.108.241:22`，用户 `cubestudio` |
| 容器（你的 node） | `172.16.108.241:<你的端口>`，用户 `root` |
| 持久化目录 | `/mnt/<你的NETID>`（=容器里的 `/`），**只有这里的数据不会丢** |
| 同时运行上限 | 每人最多 **2 个** notebook 同时 Running |
| 校外连接 | 必须先连学校 VPN：https://inc.sysu.edu.cn/service/vpn |
