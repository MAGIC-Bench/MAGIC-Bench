# 多维评估框架 — NFR 指标表（框架权威副本）

> 源自 `nfr_table_formatted.md`。本副本是框架/codex 实际读取并据以打分的版本。
> **全局计分规则：每个指标最终都是 0/1（满分=1 或 0）。**
> **比例型指标（"被正确阻断/总数"等）必须 100% 全部通过才得 1，否则 0**（不是按比例给分）。
> 静态指标由 codex 读候选代码、按本表规则判 0/1（不用机械工具）；运行时指标跑候选实测。
> 出卷时由 codex **按原仓为每个指标打标"适用/不适用"**，验卷只测打标为适用的指标。

---

## 一、安全性（SEC）

1. **SEC1 — 未授权读阻断**【运行时】
   - 测量安全性测试用例中有关未授权读操作的测试用例。
   - 得分：未授权读测试用例**全部被正确阻断（100%）才得 1，否则 0**。

2. **SEC2 — 无敏感信息硬编码**【静态】
   - 让 codex 扫描代码，检查敏感信息（如 token、密钥等）是否直接暴露在工程中不该出现的地方。
   - 得分：出现该类问题直接 0，否则 1。

3. **SEC3 — 敏感字段加密存储**【静态】
   - 数据库的敏感字段是否使用加密算法存储。
   - 得分：出现未用加密算法存储敏感信息的情况直接 0，否则 1。

4. **SEC4 — 未授权写阻断**【运行时】
   - 测量安全性测试用例中有关未授权写操作的测试用例。
   - 得分：未授权写测试用例**全部被正确阻断（100%）才得 1，否则 0**。

5. **SEC5 — SQL 注入防护**【运行时】
   - 生成 SQL 注入测试用例。
   - 得分：SQL 注入/恶意输入用例**全部被安全处理（100%）才得 1，否则 0**。

6. **SEC6 — 审计日志完备**【静态】
   - 需要生成审计日志的项目有无按照要求生成日志。
   - 得分：没有日志系统 0，否则 1。

---

## 二、兼容性（CMP）

1. **CMP1 — 共享环境可启动**【静态】
   - 静态扫描：是否存在明显硬编码端口，是否出现绝对路径写入。
   - 得分：出现上述情况则为 0，否则 1。

2. **CMP2 — 接口契约合规**【运行时】
   - 使用 schema validator 检查结构。
   - 得分：出现不符合的情况即为 0，否则 1。

---

## 三、可靠性（RLY）— 全部【运行时】

1. **RLY1 — 长时执行无故障**：重复执行标准命令。出现异常 oracle 或状态异常为 0，否则 1。
2. **RLY2 — 脏状态重启可用**：不完全清理状态下重启成功 3 次；每次重启后必须完全通过冒烟测试，否则 0。
3. **RLY3 — 数据库断连容错**：需 DB 的项目，外部断开 DB 后是否返回明确错误、主进程不崩溃。崩溃或未返回可解释错误为 0。
4. **RLY4 — 资源压力容错**：限制内存/CPU 观察状态。进程崩溃或未返回可解释错误为 0。
5. **RLY5 — 并发执行无故障**：HTTP 并发请求 / CLI 并发多进程。出现死锁/崩溃/超时为 0。
6. **RLY6 — 强制终止后可恢复**：被强制终止后能否重启并通过冒烟测试。不能完全通过为 0。

---

## 四、可维护性（MTN）— 全部【静态，codex 判】

1. **MTN1 — 无超大单文件**：是否存在过大单文件（阈值 1000 LOC）。出现则 0。
2. **MTN2 — 无跨层反向依赖**：可从路径识别层次时检查反向依赖。出现则 0。
3. **MTN3 — 无循环依赖**：构建依赖图查强连通分量。出现则 0。
4. **MTN4 — 认知复杂度合规**：计算认知复杂度。超阈值则 0，否则 1。
5. **MTN5 — README 文档完整**：codex 判 README 是否含功能简介/构建命令/运行命令等。太过简洁得 0。

---

## 五、性能效率（PERF）— 全部【运行时】

1. **PERF1 — 运行无崩溃 / OOM / 超时**：记录 OOM/超时/崩溃/断连。失败用例/总用例高过阈值则 0，否则 1。
2. **PERF2 — 尾部延迟稳定**：尾部放大率 `TAR = L99 / max(L50, 0.001)`。高于阈值则 0。
3. **PERF3 — 内存峰值合规**：运行最大内存消耗。超阈值则 0。
4. **PERF4 — 限时正确通过**（= 功能正确性的 0/1 化）：
   ```
   100, if Ncorrect/Ntotal ≥ 0.7 且 Telapsed ≤ 600 且 no_runtime_failure
   0,   otherwise
   ```
   候选在 600s 内正确通过 ≥70% 测试用例、且无超时/OOM/崩溃/框架异常/结果不可解析等运行失败 → 1，否则 0。

---

## 六、可移植性（PTB）

1. **PTB1 — 标准构建通过**【运行时/构建】：构建时缺清单/依赖不在白名单/版本不匹配 → 0。
2. **PTB2 — 无硬编码路径**【静态】：硬编码绝对路径或硬写 `/`、`\` → 0。
3. **PTB3 — 日志不写入源码目录**【静态/运行时】：把日志/缓存写进 `src/` → 0。
4. **PTB4 — 显式编码声明**【静态】：读文件/网络流/字节转换未显式指定编码（如 UTF-8）→ 0。
5. **PTB5 — 无环境变量强依赖**【静态】：运行路径强依赖 `DATABASE_URL`/`API_KEY`/`HOME`/`TMPDIR`/`PATH` 等且无默认值/配置/`.env.example`/清晰错误提示 → 0。
6. **PTB6 — 无平台强绑定**【静态】（平台不明确时可测，仅明显多平台要求时测）：无保护使用平台专属 API（`std::os::unix`、`unistd.h`、`sys.platform`、`runtime.GOOS` 等）且无条件编译/平台分支/fallback → 0。
